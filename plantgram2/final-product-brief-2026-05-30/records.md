# Records

## 메뉴 목적

기록 메뉴는 사용자가 모든 식물의 돌봄 이력을 한곳에서 확인하는 타임라인이다. 내 식물 상세가 개별 식물 중심이라면, 기록 메뉴는 전체 활동을 시간순으로 보는 회고 화면이다.

## 화면 구성

| 영역 | 설명 |
| --- | --- |
| 제목 | 기록 |
| 유형 필터 | 전체, 물, 영양제, 분갈이, 메모, 상태, 사진 |
| 타임라인 | 날짜별 기록 목록 |
| 기록 카드 | 식물명, 기록 유형, 메모, 사진, 작성 시간 |
| 편집/삭제 | 메모/상태 수정, 기록 소프트 삭제 |

## 주요 기능

- 전체 식물 기록 조회
- 기록 유형별 필터
- 사진 기록 미리보기
- 메모/상태 기록 수정
- 기록 삭제
- 최근 활동 요약과 내 식물 다이어리 흐름 연결

## 기록 유형

현재 앱에서 관리하는 대표 기록 유형은 다음과 같다.

- 물주기
- 영양제
- 분갈이
- 메모
- 상태
- 사진

## 검토 포인트

- 삭제가 실제 데이터 제거인지 소프트 삭제인지 정책과 구현이 맞아야 한다.
- 기록 유형별 필터가 Controller 상태로 관리되어야 한다.
- 기록 저장은 Repository를 통해 Firestore/Storage에 접근해야 한다.
- 사진 기록은 Storage 경로와 Firestore 문서가 서로 불일치하지 않아야 한다.

## AI 삽입 후보

기록 메뉴는 AI를 넣기에 가장 좋은 위치다. 별도 AI 탭보다 "기록을 더 쉽게 남기는 도구"로 들어가야 한다.

### 권장 UI

- 기록 탭 상단에 한 줄 자연어 입력
- 예시 플레이스홀더: "오늘 어떤 돌봄을 했나요?"
- AI 파싱 결과를 바로 저장하지 않고 초안 카드로 보여준다.
- 사용자가 식물, 기록 유형, 날짜, 메모를 확인한 뒤 저장한다.

### 예시 흐름

1. 사용자가 "몬스테라 물 줬고 새잎이 올라왔어" 입력
2. AI가 `식물: 몬스테라`, `기록: 물주기 + 상태 메모`, `메모: 새잎이 올라옴` 초안 생성
3. 사용자가 수정 또는 확인
4. 기존 기록 저장 Repository로 저장
