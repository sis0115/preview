# Account And Community

## 계정 메뉴 목적

계정 메뉴는 사용자의 프로필, 닉네임, 알림, 공개 범위를 관리하는 설정 화면이다. Plantgram2의 계정 기능은 Kakao 로그인 기반이며, 앱 운영에 필요한 최소 설정에 집중한다.

## 계정 화면 구성

| 영역 | 설명 |
| --- | --- |
| 프로필 | 아바타, 이름, 이메일 |
| 닉네임 | 닉네임 수정 |
| 알림 | 푸시 사용 여부, 알림 시간 |
| 공개 설정 | 내 식물 공개 여부 |
| 로그아웃 | Kakao/Firebase 세션 종료 |

## 계정 검토 포인트

- Auth는 Kakao-only 정책을 유지한다.
- 프로필 공개와 `myplant.is_public` 전파가 일관되어야 한다.
- 알림 설정은 UI뿐 아니라 Firestore 저장 상태와 연결되어야 한다.
- 푸시 알림은 Firebase Functions와 FCM 토큰 정책을 함께 검토해야 한다.

## 식물별 커뮤니티 목적

커뮤니티는 독립적인 대형 게시판이 아니라 식물사전 상세에서 특정 식물에 연결되는 기능이다. 이는 post-MVP 범위를 작게 유지하면서 사용자 경험을 식물 관리 맥락에 묶는 방식이다.

## 커뮤니티 화면 구성

| 화면 | 설명 |
| --- | --- |
| 게시글 목록 | 특정 식물에 대한 사용자 게시글 |
| 게시글 작성 | 식물 맥락을 가진 글 작성 |
| 게시글 상세 | 본문과 댓글 목록 |
| 댓글 입력 | 사용자 댓글 작성 |
| 댓글 수 | Firebase Functions 기반 집계 |

## 커뮤니티 검토 포인트

- 식물별 게시글만 우선 지원한다.
- 전체 커뮤니티 탭이나 범용 게시판으로 확장하지 않는다.
- 댓글 수 집계는 클라이언트 직접 갱신보다 Cloud Functions를 우선한다.
- 삭제 권한은 작성자 기준으로 제한되어야 한다.
- 공개 식물 사용자 흐름과 커뮤니티가 같은 식물 상세 맥락에서 자연스럽게 연결되어야 한다.

## AI 삽입 후보

커뮤니티에는 초기에 AI를 강하게 넣지 않는 것이 좋다. 다만 다음은 후보가 될 수 있다.

- 게시글 작성 전 내 기록 기반 요약 제안
- 댓글/게시글 신고 또는 유해 표현 보조 필터
- 같은 식물을 키우는 사용자들의 반복 질문 요약

MVP에서는 기록 AI를 먼저 안정화한 뒤 커뮤니티 보조 기능을 검토하는 순서가 적합하다.
