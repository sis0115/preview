# Menu Overview

## 하단 탭 구조

현재 `HomeTab` 기준 하단 메뉴는 4개다.

| 메뉴 | 코드 기준 화면 | 역할 |
| --- | --- | --- |
| 내 식물 | `MyPlantHome` | 보유 식물, 오늘 할 일, 일정, 다이어리 관리 |
| 사전 | `DictionaryHome` | 식물 검색, 추천 섹션, 식물 상세 정보 탐색 |
| 기록 | `RecordHome` | 전체 돌봄 기록 타임라인과 유형별 필터 |
| 계정 | `AccountPage` | 프로필, 닉네임, 알림, 공개 범위, 로그아웃 |

## 보조 진입 화면

| 화면 | 진입 경로 | 역할 |
| --- | --- | --- |
| 식물 상세 | 내 식물 목록/카드 | 개별 식물 정보와 돌봄 액션 |
| 식물 등록/수정 | 내 식물 | 내 식물 이미지, 이름, 종, 관리 정보 입력 |
| 식물사전 상세 | 사전 목록/추천 섹션 | 관리법, 상세 팁, 비슷한 식물, 커뮤니티 진입 |
| 식물별 커뮤니티 | 식물사전 상세 | 특정 식물에 대한 사용자 게시글/댓글 |
| 기록 작성/수정 | 내 식물 상세, 기록 탭 | 물주기, 영양제, 분갈이, 메모, 상태, 사진 기록 |

## 기능 우선순위

현재 앱은 식물 관리 MVP를 우선한다.

| 우선순위 | 포함 |
| --- | --- |
| 핵심 | Kakao 로그인, 내 식물, 돌봄 일정, 다이어리, 식물사전, 기록, 계정 |
| 확장 | 식물별 커뮤니티, 공개 식물 연결, 댓글 알림, 물주기 알림 |
| AI 후보 | 자연어 돌봄 기록 입력, 주간 요약, 기록 누락 감지 |
| 제외 | 쇼핑, 중고거래, 클래스, 컨설팅, 배송 컨설팅, 관리자, QR, 레거시 결제 |

## 앱 사용 흐름

1. 사용자는 Kakao로 로그인한다.
2. 내 식물을 등록하고 이미지/이름/식물종/관리 기준을 지정한다.
3. 내 식물 홈에서 오늘의 날씨, 물주기 대상, 포인트성 요약을 확인한다.
4. 일정 탭에서 예정된 돌봄을 확인하고, 다이어리 탭에서 과거 기록을 본다.
5. 식물사전에서 식물을 검색하거나 추천 섹션을 탐색한다.
6. 식물사전 상세에서 관리법과 상세 팁을 확인하고, 같은 식물 커뮤니티로 진입한다.
7. 기록 탭에서 전체 기록을 유형별로 필터링하고 수정/삭제한다.
8. 계정에서 닉네임, 알림 시간, 내 식물 공개 여부를 관리한다.

## 현재 디자인 시스템 해석

- 홈/사전/기록/계정 모두 흰색 배경을 기본으로 한다.
- 주요 강조색은 그린/민트 계열이다.
- 기능 화면은 큰 설명보다 실제 작업 버튼, 카드, 리스트를 우선한다.
- 카드형 UI는 정보 묶음 단위로 사용된다.
- 식물 이미지는 앱 분위기를 만드는 핵심 요소이며, 이미지가 없을 때는 안정적인 플레이스홀더가 필요하다.
