# My Plant

## 메뉴 목적

내 식물 메뉴는 Plantgram2의 중심 화면이다. 사용자가 키우는 식물과 오늘 해야 할 돌봄을 빠르게 확인하고, 개별 식물의 일정과 기록으로 이어지는 허브 역할을 한다.

## 화면 구성

| 영역 | 설명 |
| --- | --- |
| 상단 요약 카드 | 날씨, 물주기 대상, 내 식물 포인트/요약을 가로 카드로 표시 |
| 탭: 내식물 | 등록된 식물 목록과 카드형 정보 |
| 탭: 일정 | 예정된 물주기/돌봄 일정 |
| 탭: 다이어리 | 식물별 기록과 활동 이력 |
| 식물 등록 | 새 식물 추가, 이미지 등록, 기본 정보 입력 |

## 주요 기능

- 내 식물 목록 조회
- 식물 이미지 등록 및 표시
- 식물 상세 정보 확인
- 물주기, 영양제, 분갈이, 메모, 상태, 사진 기록
- 다이어리 유형 필터
- 최근 활동 요약
- 알림 설정과 연결되는 물주기 예정일 관리
- 공개 설정이 켜진 식물은 사전 상세의 "이 식물을 키우는 사람들" 흐름에 연결

## 기존 Plantgram1에서 가져와야 하는 의도

레거시의 모든 메뉴를 이식하는 것이 아니라, 식물 관리에 직접 필요한 기능만 가져오는 방향이다. 특히 백엔드 SQL에서 확인 가능한 식물 관리 주기, 기록 유형, 식물 데이터 구조는 Firebase 모델과 Repository 계층으로 재해석해야 한다.

## 검토 포인트

- 화면에서 Controller만 호출하고 Firebase 접근은 Repository에 있는지 확인해야 한다.
- D-day/물주기 예정일 계산이 중복되지 않고 공통 유틸 또는 도메인 헬퍼로 정리되어야 한다.
- `MyplantController`가 목록/상세/수정 책임을 과도하게 섞고 있는 리스크는 계속 관리해야 한다.
- 이미지 업로드와 공개 범위 변경이 Firestore 데이터와 일관되게 반영되어야 한다.

## AI 삽입 후보

내 식물 상세 또는 기록 작성 액션시트 상단에 자연어 입력을 배치하는 것이 가장 자연스럽다.

예시:

> "스투키에 물 줬고 잎 끝이 조금 마른 것 같아"

AI는 이 문장을 식물명, 기록 유형, 상태 메모, 날짜 후보로 변환하고 사용자가 확인 후 저장한다.
