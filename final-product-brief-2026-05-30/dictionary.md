# Dictionary

## 메뉴 목적

식물사전은 사용자가 식물을 검색하고, 추천 섹션을 통해 새로운 식물을 탐색하며, 상세 관리법을 확인하는 화면이다. Plantgram2에서는 내 식물 등록과 기록 흐름을 보조하는 지식 기반 역할을 한다.

## 화면 구성

| 영역 | 설명 |
| --- | --- |
| 검색창 | 식물명 검색 |
| 오늘의 식물 | 대표 식물을 카드로 강조 |
| 추천 섹션 | 초보 추천, 공기정화, 인테리어, 반려동물 주의 등 |
| 식물 목록 | 사전 데이터 기반 식물 리스트 |
| 상세 화면 | 기본 정보, 설명, 관리 정보, 상세 팁, 유사 식물 |
| 식물별 커뮤니티 진입 | 특정 식물에 대한 게시글 목록으로 이동 |

## 현재 추천 섹션

코드 기준으로 확인되는 주요 섹션은 다음과 같다.

- 초보 추천
- 공기정화
- 인테리어
- 반려동물 주의
- 아이 주의
- 물을 좋아해요
- 햇빛을 좋아해요
- 따뜻한 곳 추천
- 최근 업데이트
- 인기 식물

## 상세 화면 기능

- 식물 이미지, 이름, 타입 표시
- 기본 정보와 설명
- 관리 정보
- 추천/주의 라벨
- `plant_tips` 기반 상세 관리 팁
- 비슷한 식물 추천
- 같은 식물을 키우는 사용자 흐름
- 식물별 커뮤니티 진입

## 기존 Plantgram1에서 가져와야 하는 의도

레거시 백엔드 SQL의 식물 데이터, 분류, 관리 기준을 검토해 Firestore 사전 데이터와 매핑해야 한다. 단순 화면 복제가 아니라 검색/추천/상세 관리 기준이 실제 식물 관리 흐름과 연결되어야 한다.

## 검토 포인트

- 사전 목록이 Firestore 우선으로 동작하는지 확인한다.
- 빠른 추천 섹션 로직이 View에 흩어지지 않고 Controller/Repository 흐름에 있는지 확인한다.
- 상세 팁과 유사 식물 추천이 빈 데이터에서도 깨지지 않아야 한다.
- 식물별 커뮤니티는 전체 커뮤니티 탭이 아니라 식물 상세 맥락에서 진입하는 구조를 유지한다.

## AI 삽입 후보

사전에는 AI 상담 챗봇보다 보조형 기능이 적합하다.

- 검색어 오타 보정
- 사용자의 내 식물/환경에 맞춘 추천 사유 설명
- 사전 상세에서 "내 식물 기록 기준으로 주의할 점" 요약

단, MVP에서는 질병 진단이나 이미지 판별을 넣지 않는 것이 현재 기획 방향과 맞다.
