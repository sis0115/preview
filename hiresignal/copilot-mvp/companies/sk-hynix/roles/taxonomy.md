# SK하이닉스 직무 Taxonomy 리서치

생성: 2026-06-03

## 결론

하이닉스 데이터는 추가 영상만 계속 쌓기보다, 공식 직무 체계를 기준으로 현재 자막 데이터를 재분류하는 것이 더 효과적이다.

이유:

- H1에서 수집한 공식 직무 영상은 실제 채용 직무 체계와 잘 맞는다.
- 현재 추출기는 여러 영상을 `SK하이닉스 / 반도체 직무`로 뭉뚱그린다.
- 제품 가치가 생기려면 지원자가 `양산기술`, `R&D 공정`, `소자`, `설계`, `P&T`, `Maintenance` 중 어디에 가까운지 먼저 판별해야 한다.

## 2026/최근 채용공고 관점

외부 보도와 SK Careers 공고를 보면 SK하이닉스는 설계, 소자, R&D 공정, 양산기술 등을 포함해 다수 직무를 모집한다. 2024 Junior Talent 공고에는 R&D 공정, R&D 장비, PKG 개발, 소자, 기반기술, DRAM 설계, HBM Digital 설계, NAND 설계, SoC 설계 및 검증, Product Engineering, Application Engineering, Software, 양산기술, 양산기술(P&T), 양산관리 등이 명시되어 있다.

이걸 제품에 반영하면 `하이닉스 지원`을 하나로 묶으면 안 된다. 최소한 아래 선택지가 있어야 한다.

- Product Engineering

- 양산기술
- 양산기술(P&T)
- 양산관리

## 제품적으로 강화할 방식

### 1. 먼저 직무 선택 질문을 넣는다

사용자에게 바로 자소서를 받기보다, 하이닉스 지원 직무를 먼저 선택하게 해야 한다.

- 설계
- 소자
- R&D 공정
- Product Engineering
- PKG 개발
- 양산기술
- 양산기술(P&T)
- 설비관리/Maintenance
- 품질보증
- 기반기술
- Utility기술
- 아직 모름

`아직 모름`이면 전공/프로젝트/경험 키워드로 추천한다.

### 2. 경험을 직무별로 다르게 번역한다

같은 프로젝트라도 직무별 번역이 달라야 한다.

예:

- 데이터 분석 프로젝트 -> 양산기술: 수율/품질 개선, 공정 조건 최적화
- 데이터 분석 프로젝트 -> R&D 공정: 변수 통제, 실험 설계, 원인 규명
- 데이터 분석 프로젝트 -> Product Engineering: 제품 평가, 테스트 결과 분석, 개선 방향
- 장비 프로젝트 -> Maintenance: 고장 대응, 예방 정비, 성능 최적화
- 화공/전기/플랜트 경험 -> Utility기술: 안정적 공급, 클린룸/설비 운영, 안전/환경

### 3. 하이닉스 전용 라벨러가 필요하다

현재 일반 추출기는 `SK하이닉스 / 반도체 직무`로 많이 뭉갠다. 아래 키워드 기반으로 1차 라벨을 강제하는 것이 낫다.

| 라벨 | 강제 키워드 |
|---|---|
| 양산기술 | 양산기술, 생산성, 수율, 장비와 공정 조건, 양산 데이터 |
| R&D 공정 | R&D 공정, 공정 개발, 분석 장비, 변수, 실험, 원인 |
| 양산기술(P&T) | P&T, Package, TEST, PKG, 후공정, 테스트 |
| 소자 | 소자, 디바이스, 성능, 설계와 공정 사이 |
| 설계 | 설계, 회로, 아키텍처, 검증, DRAM, HBM Digital |
| Maintenance | Maintenance, 메인트, 설비관리, 고장 대응, 유지보수 |
| Product Engineering | PE, 평가, 분석, Test Solution, 제품 완성도 |
| 품질보증 | 품질보증, 고객 품질, 인증, 불량, 변경관리 |
| 기반기술 | 계측, 분석, Mask, 소재, 장비, 기반기술 |
| Utility기술 | Utility, 클린룸, Gas, Chemical, UPW, 공조, 배관, 전기 |
