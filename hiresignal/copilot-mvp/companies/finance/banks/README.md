# Bank-Specific Copilot Skeleton

금융권은 은행별 채용 특색을 유지한다.

지금 금융권은 메인 타겟이 아니므로 깊게 수집하지 않는다. 대신 각 은행 폴더에 다음 뼈대를 둔다.

- `README.md`: 은행별 코파일럿 포지션
- 나중에 추가할 파일:
  - `copilot_kb.md`: 은행별 최종 지식 베이스
  - `copilot_kb.json`: 은행별 코파일럿 주입용 JSON
  - `judgment.md`: 데이터 품질과 수집 우선순위 판단
  - `sources/`: 공식 채용공고, 직무기술서, 면접/설명회 근거
  - `collections/`: 유튜브/후기/공고 수집 배치

## 은행별 초기 포지션

| 은행 | 코파일럿 셀링 포인트 | 우선 보강 데이터 |
|---|---|---|
| IBK기업은행 | 중소기업금융, 정책금융, 공공성, 디지털/IT | 중소기업금융 면접, 금융일반/디지털/IT JD |
| 우리은행 | 기업금융, 개인금융, 비전공자 직무 연결 | 기업금융 면접, 개인RM, PT/임원면접 |
| 하나은행 | PT, BEI, 하나FIT, 조직 적합성 | PT/BEI 기출, 가치관/조직 적합성 |
| KB국민은행 | UB, PB/자산관리, ICT | UB/PB/ICT 채용공고, 면접 방식 |
| 신한은행 | 디지털/ICT, 고객경험, 플랫폼 금융 | 디지털/ICT JD, 앱/데이터/플랫폼 사례 |
| NH농협은행 | 지역성, 농업/공공성, 일반행원, 디지털 전환 | 지역/농업금융, 일반/디지털 면접 |

## 제품 적용

금융권 공통 KB는 기본 질문지다.

은행별 KB는 다음 질문에 답해야 한다.

- 왜 이 은행인가
- 이 은행의 주력 고객/사업은 무엇인가
- 이 은행의 채용 전형에서 강하게 보는 면접 유형은 무엇인가
- 같은 고객 응대 경험이라도 이 은행에서는 어떤 언어로 바꿔야 하는가
- 디지털/IT/기업금융/PB 등 트랙별로 다른 준비 포인트는 무엇인가
