# Finance Copilot Data

금융권 취업 코파일럿 데이터 폴더다.

금융권은 `은행권 공통 구조 + 은행별 채용 특색`으로 관리한다.

중요한 제품 포인트:

- 금융권도 회사별 코파일럿을 유지한다.
- 다만 지금 타겟은 아니므로 깊게 수집하지 않고, 은행별 폴더 뼈대만 만든다.
- 공통 KB는 은행권 기본 질문지로 쓰고, 실제 제품에서는 목표 은행을 먼저 물어 은행별 KB로 분기한다.

## 대표 파일

- `copilot_kb.md`: 금융권 취업 코파일럿 지식 베이스
- `copilot_kb.json`: 코파일럿 주입용 구조화 데이터
- `judgment.md`: 데이터 품질과 계속 수집 여부 판단

## 폴더

- `roles/`: 은행권 직무/면접 taxonomy
- `sources/`: 공식 채용/정책/산업 출처
- `banks/`: IBK, 우리, 하나, KB, 신한, 농협 등 은행별 채용 특색
- `collections/youtube_batch2a/`: 기존 금융권 유튜브 수집 배치
- `raw/`: 내부 분석 원본

## 현재 판단

금융권 코파일럿의 공통 핵심은 `고객 응대 경험`, `은행 관심 증거`, `필기 루틴`, `면접 유형별 훈련`이다.

반도체/자동차처럼 기술 직무 depth만 깊게 파는 방식은 맞지 않다. 금융권은 자소서 문장이 면접 꼬리질문을 버틸 수 있는지, 고객 상황을 신뢰와 정확성으로 처리할 수 있는지, 은행별 PT/BEI/세일즈 구조에 맞는지가 중요하다.

## 은행별 분기 원칙

코파일럿은 금융권 지원자에게 다음 순서로 질문한다.

1. 목표 은행
2. 지원 트랙: 일반, 지역인재, 디지털, IT, PB, 기업금융 등
3. 은행별 면접 유형: PT, BEI, 세일즈, 롤플레잉, 가치관, 기업금융 case
4. 은행별 차별 키워드: 중소기업금융, 디지털, 지역성, 자산관리, 플랫폼, 공공성 등

현재는 은행별 구조만 열어두고, 실제 확장은 타겟이 될 때 진행한다.
